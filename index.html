Year: 2017

Handies audio player

A spinoff project from the previous one, was about the need from the operators to have all handies communication within an app, in a way they were able to reproduce them as a music player (Forward, Backward, Play, Pause, Stop).

Because of the nature of the servers behind this communication network, there were many technicals challenges to overcome (sometime that finally did it successfully).

One of the key solution to make this work, was about creating a audio player as a clickable timeline, using HTML Canvas to render and visualize all the needed information (as an audio mixer timeline). I dedicated 3 months of work dedicated only to make this work.

The tecnology stack (I participated in the 3 layers) were:

  • Front-End: developed using TypeScript with Angular. Because of the special needs for the app, a big part also was developed with Vanilla JavaScript and the natives HTML5 elements.
  • Back-End: was built as an RESTful API developed with .Net Core 2.0 (the latest at that time). With Token JWT as security layer.
  • Database: was SQL Server with SQL Service Broker for communications between servers.

Development process

  • Team of 4 developers, 1 tester.
  • Daily meetings.
  • TFS as Source Control.
